## 2. 这本书的设计指纹

这本的核心价值不是具体人物、家族、功法或道具，而是一个可迁移的商业叙事引擎：

> 旧世界专业体系进入新世界后，主角用另一套知识框架解释、破解、改造当地规则，从而获得连续的身份反差、资源收益和权力牵引。

### 类型定位

- 主类型：穿越异界爽文
- 副类型：魔幻大陆、旧世界专业体系、强者误判、权力势力线
- 结构类型：单主角漫游推进 + 阶段性势力卷
- 情绪基调：轻佻调侃、猎奇、低压爽点、偶发悬疑

### 核心发动机

每章大体围绕这个循环变化：

```text
异界规则/强者/势力出现
-> 主角用旧世界专业体系、术法、符号系统、现代常识或错位理解进行判断
-> 当地人误判主角等级或身份
-> 主角以低成本获得资源、关系、秘密或威慑
-> 引出更高层级的势力、神秘身份或规则冲突
```

### 读者承诺

这本给读者的不是严肃升级，而是：

- 主角看似弱者，实际有外部体系优势。
- 当地强者按本世界规则判断主角，因此频繁误判。
- 主角把魔法、剑士、魔兽、神教、王权等当地秩序当成可研究、可借用、可戏弄的对象。
- 新地图/新势力不断打开，维持“还有什么怪东西”的猎奇感。

## 3. 状态变量设计

如果把这本蒸馏成 BestSeller 可用的题材机制，应抽取这些状态变量，而不是抽取具体剧情。

| 状态变量 | 说明 | 适合进入系统的位置 |
|---|---|---|
| 跨体系理解度 | 主角对当地魔法、剑士、神教、契约等规则的理解进度 | story grammar / kernel |
| 旧体系可用度 | 旧世界专业体系、符号技术、术法在新世界的适配程度 | power_systems |
| 真实实力隐藏度 | 外界对主角能力的误判程度 | chapter state |
| 绑定资产 | 坐骑、奴仆、盟友、神器、契约关系等可支配资源 | state ledger |
| 政治牵连度 | 王子、皇权、家族、神教等势力对主角的拉扯 | plot tree |
| 神秘来源线 | 旧世界符号、穿越痕迹、神权背景等长期谜团 | setup/payoff |
| 情欲/猎奇噪声 | 早期网文常见卖点，现代系统应限制使用 | anti-cliche gate |

## 5. 可复用机制

以下机制可以进入素材库或题材语法。

### plot_patterns

1. 跨体系规则套利  
   主角携带旧世界专业知识进入新世界后，不直接按当地升级路线竞争，而是用旧体系解释、破解、绕开当地规则。

2. 弱身份强能力误判  
   主角的新身份在当地社会被判定为弱者，但他实际拥有不在当地评估体系内的能力。

3. 宿主身份债  
   穿越者继承原身身份后，必须承担原身家族、仇怨、契约和社会义务，不能无限漫游。

4. 高阶存在资产化  
   高阶魔兽、异族、强者不是一次性敌人，而是被转化为坐骑、护卫、债务人、盟友或势力入口。

5. 小地图到大势力递进  
   野外生存 -> 城镇商业 -> 王权战争 -> 家族清算 -> 神权/黑暗终局。

### scene_templates

1. 初到异界规则盘点  
   主角通过原身记忆、当地书籍、第一次冲突快速识别世界基础规则。

2. 当地强者误判主角  
   对手按本世界等级体系判断主角，忽略主角外部体系能力。

3. 专业能力低成本破局  
   主角用旧世界专业技能、符号系统、阵式逻辑、现代常识或另一种技术，在对手没理解前完成反制。

4. 社会场景展示规则  
   酒店、赌场、佣兵工会、魔法工会、夜市、宫廷晚宴等场景承担世界规则展示，不只是过场。

5. 绑定资产带来新麻烦  
   每获得一个强者、神器、奴仆或势力关系，都同步增加被追查、被利用或被卷入政治的风险。

### reader_rewards

- 身份反差爽
- 专业知识碾压
- 强者误判
- 异界猎奇
- 资源收编
- 势力借力
- 轻喜剧式冒犯权威

## 6. 禁止复用清单

以下内容只能作为“来源样本的独有表达”记录，不能进入生成提示作为可复用素材。

- 书名、作者名、原角色名、原家族名、原大陆/国家名。
- 具体功法、法宝、道具和坐骑名称。
- “电脑白光/雷击穿越”这类具体开局桥段。
- “被兄弟推下悬崖后借尸还魂”的具体组合。
- 具体章节标题、桥段顺序和人物关系。
- 早期网文中明显不适合现代产品调性的女性物化桥段，应进入反套路和安全约束，不应作为卖点复用。

## 7. 拟导入 BestSeller 的结构

### 7.1 Material Library

首批只建议导入 `status=review` 的抽象条目，人工审核后再激活。

建议维度：

- `plot_patterns`
- `scene_templates`
- `power_systems`
- `character_archetypes`
- `anti_cliche_patterns`
- `thematic_motifs`

样例文件：

- `data/distillation/source-0001/material_entries.sample.jsonl`

### 7.2 Story Design Grammar

建议新增或扩展一个题材语法：

```yaml
key: otherworld-cross-system
name: 异界跨体系规则套利
state_variables:
  - cross_system_understanding
  - local_power_misread
  - old_system_adaptation
  - bound_assets
  - political_entanglement
  - source_world_mystery
chapter_change_vectors:
  - learn_local_rule
  - exploit_rule_gap
  - convert_enemy_or_resource
  - reveal_higher_authority
  - pay_identity_debt
reader_rewards:
  - weak_identity_strong_core
  - knowledge_arbitrage
  - power_misread_payoff
  - exotic_world_discovery
  - authority_prank
forbidden_defaults:
  - copy_named_artifact
  - copy_named_family_conflict
  - female-objectification-as-core-reward
  - effortless-harem-collection
  - one-way-power-fantasy-without-cost
```

### 7.3 Story Design Kernel 注入方式

新书立项时，不要提示“像这本书”。应该提示：

```text
采用“跨体系规则套利”引擎：
- 主角来自 A 世界，掌握一个专业体系；
- B 世界有一套看似强大的主流规则；
- 主角的专业体系能解释 B 世界的一部分盲区，但不是万能；
- 每次套利都带来可见收益，也制造更高层级注意；
- 禁止复用样本书的人物、地名、功法、道具和桥段。
```

## 8. 质量门设计

蒸馏结果进入系统前，必须通过这些门：

| 门 | 检查 |
|---|---|
| 抽象度门 | 条目是否只描述机制，而不是复述原剧情 |
| 专有名词门 | 是否包含来源书角色、地名、功法、道具名 |
| 相似度门 | 新生成项目是否与来源书在开局、身份、资产组合上过近 |
| 现代调性门 | 是否把过时的性别/奴役/物化桥段当作核心奖励 |
| 成本门 | 主角每次跨体系套利是否带来新代价或新注意 |
| 状态门 | 每章至少改变一个状态变量，而不是只展示猎奇 |

## 9. 用这本书反向生成的新书方向

这是蒸馏后的可迁移方向，不是复刻：

1. 法医闯修仙界  
   用病理、毒理、痕检破解丹毒、魂伤、宗门命案。

2. 建筑师闯魔法城邦  
   用结构力学、城市规划、消防系统破解魔法塔和贵族城区。

3. AI 工程师闯神术帝国  
   用系统建模、信息论、博弈论拆解神谕、契约和教会权力。

4. 中医/药师闯蒸汽异界  
   用经络、药性、体质辨证与炼金医学互相校验。

共同核心是“专业体系跨世界套利”，不是来源样本的原始职业、异界设定或原桥段。
