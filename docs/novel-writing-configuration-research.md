# 小说写作配置与提示词强化研究

## 目标

这份说明回答三个问题：

1. 为什么要把“写作配置”扩成结构化 `writing_profile`
2. 为什么 prompt 需要显式加入平台化连载约束
3. 这些配置和提示词强化，如何映射到当前系统

## 参考来源

### 开源写作项目

- `novelWriter`
  - <https://github.com/EdwardAThomson/NovelWriter>
  - 重点启发：长篇创作需要把项目、章节、人物、地点、笔记拆成稳定对象，而不是单一文档。
- `Manuskript`
  - <https://github.com/olivierkes/manuskript>
  - 重点启发：角色、世界、情节、章节结构应当是独立可维护的数据对象。

### 国内网文平台/写作指导

- 番茄作者专区：如何设计吸引读者的开篇
  - <https://fanqienovel.com/writer/zone/article/7480087779494346776>
- 番茄作者专区：长篇写作中的节奏与持续追读
  - <https://fanqienovel.com/writer/zone/article/7517213257795387416>

这些官方作者向内容反复强调几个点：

- 开篇必须尽快建立“读者承诺”
- 主角卖点和差异化必须在早期明确亮出
- 信息释放不能靠大段说明，要靠事件、冲突和结果
- 章节末必须有钩子，保证读者继续追读
- 长篇不是一次性写完，而是持续连载推进，需要稳定节奏和兑现体系

## 研究结论

### 1. “题材”不够，必须有“平台化写作画像”

仅有 `genre / target_word_count / target_chapters`，只能决定大方向，不能决定作品是否具备平台可读性。

真正影响成品效果的，是这些更细的决策：

- 作品投向哪个平台
- 对读者的承诺是什么
- 卖点是什么
- 依赖哪些常见套路
- 主角的 archetype 是什么
- 是否有金手指 / 信息差 / 血脉 /系统 / 特殊资源
- 爽点兑现是短回报还是长回报
- 每章尾钩靠秘密、利益、危机还是反转
- 背景设定是硬规则还是软氛围
- 语言风格偏利落还是细腻

因此当前系统把这些拆成 `writing_profile`，而不是让它们散落在 `metadata` 或 prompt 文本里。

### 2. 长篇好看，关键不是“多设定”，而是“兑现承诺”

商业连载小说不是“世界观百科全书”。更有效的写法是：

- 开篇先给异常
- 立即给主角差异化优势
- 很快给一个短期目标
- 在前几章内兑现第一轮收益/反杀/发现
- 再把更大的阴影、真相或更高层级冲突抬出来

所以 prompt 不能只说“写精彩一点”，而要显式要求：

- 前三章完成什么
- 当前场景必须推进什么
- 结尾必须留下什么
- 设定解释必须如何释放

### 3. 审校必须评“平台适配”，不能只评语法和连贯性

如果 review 只看：

- 连贯
- 字数
- 设定一致

那写出来的内容往往“合理但不好追”。因此当前 review/rewrite prompt 也需要加入：

- 读者承诺是否兑现
- 卖点是否清楚
- 钩子是否够强
- 开篇是否足够快
- 本章是否真的让读者想继续看

## 对当前系统的映射

### 新配置层

当前新增了结构化 `writing_profile`，包含：

- `market`
- `character`
- `world`
- `style`
- `serialization`

并提供按题材自动预设。

### 新 prompt 层

以下 prompt 现在都会读取 `writing_profile`：

- `book_spec`
- `world_spec`
- `cast_spec`
- `volume_plan`
- `chapter_outline_batch`
- scene draft
- scene review/rewrite
- chapter review/rewrite

### 新追踪层

项目创建和自动写作之后，页面与 API 可以直接查看：

- 项目摘要
- 实际生效的 `writing_profile`
- workflow 总数
- 最近 workflow 状态
- 每条 workflow 的 step 详情

## 当前仍可继续优化的方向

### 1. 平台模板更细

现在只做了题材预设，后续可以继续加入：

- 番茄男频
- 番茄女频
- 起点升级流
- 晋江成长/言情
- 悬疑推理
- 轻小说风格

### 2. 质量评分继续细化

下一步可以把这些维度进一步结构化入 `quality score`：

- 开篇抓力
- 主角辨识度
- 爽点兑现密度
- 信息释放效率
- 尾钩强度
- 配角功能性
- 世界规则使用率

### 3. 引入样书对比评测

系统已经有 benchmark 能力，后续可以把不同 `writing_profile` 和 prompt 版本拉进统一评测，对比：

- 追读感
- 结构稳定性
- 信息密度
- 审校通过率

## 当前建议

如果目标是写一本更像“成品网文”的小说，不要直接只改 premise。更有效的做法是：

1. 先定 `writing_profile`
2. 再跑规划
3. 看故事圣经和叙事图谱是否真正承接了卖点
4. 再看 scene/chapter review 是否在持续检查平台适配和钩子强度

否则系统仍然会写出“结构完整但不够好追”的内容。
