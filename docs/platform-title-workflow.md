# 平台化书名生成规范与独立工作流

生成日期：2026-05-17  
适用范围：BestSeller 的上架资料、推荐位测试、同一故事跨平台改名。

## 1. 调研结论

平台书名不是“好听”问题，而是分发入口问题。不同平台的标题承担的任务不同：

| 平台 | 当前榜单/规则观察 | 书名主倾向 | 生成策略 |
| --- | --- | --- | --- |
| 番茄小说 | 官方榜单按分类和在读数据排榜，阅读榜要求 30 万字以上、签约、未下架、已推荐；新书榜要求 30 万字以下、签约、已推荐且未断更。页面显示榜单数据每日更新到前一日 24:00。 | 移动推荐流优先，一眼看懂“开局事件 + 主角选择 + 爽点承诺”。常见 `开局...`、`我在...`、`让你...你...`、`全民...`。 | 先生成可秒懂强钩子名，再保留 1-2 个短 IP 名。标题可用冒号、逗号、问号。 |
| 起点中文网 | 官方榜单包括月票、畅销、阅读指数、推荐、收藏等；搜索结果样本中高位书名有《捞尸人》《没钱修什么仙？》《玄鉴仙族》《苟在初圣魔门当人材》等。起点社区公约明确“标题”等信息内容也受合规管理。 | 更接受概念化、世界观化、职业化、家族/制度化短标题。标题可有文学感和 IP 感，不必把爽点全写完。 | 生成“短 IP 名 + 设定质感名 + 反常识短句”三组，避免番茄式过度口播。 |
| 七猫中文网 | 大热榜页面标明“基于昨日书籍热度排行”，2026-05-17 抓取样本包括《盖世神医》《警报！真龙出狱！》《逍遥四公子》《权力巅峰：从省府秘书开始》《鸿蒙霸体诀》《葬神棺》等。作品登载规则要求遵守发布规则、禁止违法违规和侵权内容。 | 标题更直给身份、地位、逆袭路线、强职业或强物件。男频偏神医、权力、边关、玄幻器物；女频偏身份反转、豪门/总裁/婚恋。 | 使用“身份/职业 + 起势”“X 巅峰：从 Y 开始”“强物件/功法名”结构。 |
| 晋江文学城 | 原创金榜和月度榜样本中既有《新聘》《独白》这类短标题，也有《警察，但犯罪大师系统[刑侦]》《协议恋爱，但大佬失忆了》《我真不是狼王的心尖宠！》这类关系/人设钩子。简易审核标准把作品名称、简介、一句话简介、标签等文案纳入违规原因提示。 | 人物关系、情绪张力、设定标签优先。短标题要有气质，长标题要有人设冲突；方括号标签是重要筛选信号。 | 生成“短情绪名 + 关系钩子名 + 标签补强名”，不要套男频强爽标题。 |
| 飞卢小说网 | 榜单样本高度口播化，如“综武：...”“娱乐：...”“名义：...”“盘点...”“让你...你...”等。 | 频道前缀、同人宇宙、开局身份和爽点结果必须前置。 | 标题可长，可强类型，可像视频标题。 |
| 塔读/纵横类 | 塔读书库样本中有《代晋》《仙妻如云》《我在乱世娶妻长生》《天启神图》《穿越刘备，开局硬刚曹操》等。 | 介于传统类型名和免费阅读直观卖点之间，历史/玄幻/都市可更稳重。 | 生成传统 IP 名、器物/时代名、身份起势名。 |

## 2. 硬规则

1. 书名、简介、标签、封面文字都属于平台审核范围；标题不能规避违法违规、低俗、侵权、冒用身份、夸大误导等内容。
2. 书名必须与正文前三章承诺一致。不能用“神医”“系统”“上交国家”“犯罪大师”等标题词，但正文没有兑现。
3. 同平台上架前必须做重复/近似重复检查。番茄公开有未签约断更作品书名资源重置机制，说明书名资源和唯一性会影响作者体验。
4. 封面若含书名，封面书名必须与作品信息一致；修改书名后需要同步封面和简介。
5. 标题生成不直接复刻榜单书名，只抽取结构：入口、长度、语气、标签位置、类型词。

## 3. 平台画像

### 番茄

推荐长度：9-22 字，允许到 30 字。  
候选结构：

- `开局{事件/金手指}，我选择{动作/立场}`
- `我在{场景}靠{物件/能力}{结果}`
- `让你{低位任务}，你{高位结果}？`
- `{类型}：从{低位身份}开始{升级动作}`

适合：免费流量、下沉读者、强节奏爽文、脑洞、短视频转化。  
不适合：只给“长夜”“归墟”“问道”这类抽象意象。

### 起点

推荐长度：3-12 字，允许到 18 字。  
候选结构：

- `{职业/身份}人`
- `{家族/制度/器物}仙族`
- `没钱修什么{核心道途}`
- `苟在{势力/世界}当{身份}`
- `{核心物件}巡天 / {核心规则}纪事`

适合：设定党、长线升级、精品付费、可 IP 化书名。  
不适合：把简介压成 25 字以上的广告句。

### 七猫

推荐长度：5-18 字，允许到 24 字。  
候选结构：

- `{强职业}下山 / {强身份}归来`
- `{爽点}巅峰：从{低位入口}开始`
- `{场景}第一{身份}`
- `{核心物件}霸体诀 / {神医/战神/权力/边关}类`

适合：移动端快节奏、身份逆袭、医武、官场、边军、玄幻。  
不适合：看不出分类和主角收益的纯文学名。

### 晋江

推荐长度：3-16 字，允许到 24 字。  
候选结构：

- `{关系/事件}倒计时`
- `协议{关系}，但{反转}`
- `{身份}如何拯救{对象}`
- `你就是那个{身份}？`
- `{短标题}[穿书/刑侦/六零/ABO/GL]`

适合：言情、纯爱、百合、无 CP、女性向悬疑、快穿、年代、同人。  
不适合：强行“开局爆杀/封神/成王”。

## 4. 独立工作流

书名生成必须从简介生成中拆出，单独执行：

1. **输入故事 DNA**：`primary_category`、`secondary_category`、`tags`、`logline`、主角身份、金手指/核心物件、读者承诺。
2. **选择平台画像**：`fanqie/qidian/qimao/jinjiang/feilu/zongheng/tadu/general`。
3. **生成平台矩阵**：默认一次生成 `fanqie/qidian/qimao/jinjiang/feilu/zongheng/tadu/general` 8 组，每组 5 个候选。
4. **硬过滤**：长度、空泛词、重复标题、违规词、承诺错配、平台错配。
5. **打分排序**：平台结构命中、移动端理解、类型覆盖、记忆度、合规风险。
6. **输出候选包**：默认 40 个候选，每个候选包含 `title/subtitle/angle/recommendation/platform_label/scope_label/display_label/pattern/score/fit_notes`。
7. **人工锁定主标题**：上架前从候选包中选择主推、备选、广告测试、短视频测试四类。
8. **同步上架资料**：选中标题后重新生成封面文字、简介首句、标签顺序和推荐位宣传语。
9. **平台回收**：上架后把点击率、加入书架率、完读率、签约/推荐结果回写到平台画像。

## 5. 当前代码落地

已新增独立服务：

- `src/bestseller/services/platform_title_workflow.py`
- `build_platform_title_workflow(profile, target_platform=...)`

`book_listing` 现在会把生成结果放入：

- `profile["title_candidates"]`
- `profile["title_workflow"]`
- `profile["legacy_title_candidates"]`：只保留历史候选用于追溯，不再作为当前展示候选

这让书名候选可以单独展示、单独审核、单独 A/B，而不是混在简介生成里。

### 全量执行规则

1. 所有新小说的候选书名都由 `build_platform_title_workflow()` 生成。
2. 候选包按平台分组输出：番茄、起点、七猫、晋江、飞卢、纵横、塔读、全平台各 5 个。
3. 每条候选都有 `platform_label`、`scope_label`、`display_label`，前端必须展示“平台/范围”，不能只显示目标平台。
4. `candidate_policy=platform_matrix_5_each` 时，`candidate_count` 应为 40，`platform_groups` 应为 8 组。
5. 历史 `title-candidates.csv` 不再覆盖当前候选，只进入 `legacy_title_candidates` 作为审计记录。
6. 前端“重新生成候选书名”不再请求大模型自由生成，而是刷新平台化书名工作流。
7. 当前输出目录可用脚本刷新：

```bash
PYTHONPATH=src .venv/bin/python scripts/regenerate_platform_title_workflows.py output
```

脚本会写出：

- `listing/title-workflow.json`
- `listing/title-candidates.csv`
- 若已有旧 CSV，则首次保留 `listing/title-candidates.legacy.csv`
