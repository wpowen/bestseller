# 七猫签约经验规则汇总

来源：[七猫作者交流区「拒稿到签约：过稿经验分享（26篇帖子）」](https://bbs.qimao.com/column/6859f2539995c)

本文只保留可执行规则，不复述原文。核心结论：七猫过稿不是单点文笔问题，而是「平台适配 + 开篇留存 + 主角驱动 + 爽点回报 + 可持续更新」同时达标。

## 1. 平台适配优先

七猫偏移动端无线风：节奏要快，情绪要明确，爽点要密，但不能像新媒体文一样只靠夸张情绪堆砌。起点慢热、纯设定流、纯文学腔、外站悬念付费逻辑都要重写成七猫读者能立即进入的形式。

执行规则：

- 动笔前先扫同平台同题材榜单，至少拆 5 本前三章。
- 选安全题材和成熟框架，在套路内微创新。
- 主角困境、差异化优势、短期目标必须早显。
- 世界观、背景、人物关系必须嵌入行动和冲突中释放。

## 2. 第一章是签约筛选器

多篇经验反复指向同一件事：黄金三章已经压缩成黄金一章。第一章不能只负责铺垫，它必须同时完成吸引、代入、期待。

硬门槛：

- 前 100 字：主角进入聚光灯。
- 前 200 字：出现可感冲突、危险、误会、侮辱、背叛或利益冲突。
- 前 600 字：读者能看出核心矛盾。
- 前 2000 字：形成情绪钩子。
- 前 6000 字：主线目标、障碍、行动方向清晰。
- 前 10000 字：证明核心循环可持续。

## 3. 前三章公式

- 第 1 章：立冲突，勾情绪，让读者知道主角被什么压住。
- 第 2 章：亮差异化优势或金手指，明确短期目标和反打方向。
- 第 3 章：完成第一次小爽点或小爆点，同时留下下一轮钩子。

爽点不是只等于打脸。任何正反馈、成长、收益、地位变化、关系推进、真相揭露，只要让读者获得情绪价值，都可以算爽点。问题在于它必须可见、及时、服务主线。

## 4. 常见拒稿理由转规则

- 主线不清：补「目标、障碍、行动、回报」。
- 代入弱：用主角视角写感官、动作、选择和后果。
- 情节平：加入冲突、信息差、短期目标和阶段回报。
- 爽点弱：让主角在 1-3 章内完成一次可感正反馈。
- 拖沓流水账：删掉不改变目标、阻碍、信息和关系的内容。
- 人设没亮点：给主角鲜明欲望、行动风格、记忆标签。
- 平台错配：回到七猫同题材榜单，复刻节奏骨架后再创新。
- AI 腔：AI 只能辅助构思，正文要保留具体动作、人物口气和作者判断。

## 5. 框架落点

已落入：

- `config/writing_methodology.yaml`：新增 `qimao_signing_gate`，保存平台适配、开篇门槛、拒稿映射和每章循环。
- `src/bestseller/services/methodology.py`：新增七猫签约约束读取与 prompt 渲染。
- `src/bestseller/services/drafts.py` / `src/bestseller/services/reviews.py`：当写作画像平台目标为七猫时，草稿和重写阶段自动注入七猫签约门槛。
- `tests/unit/test_methodology_qimao.py`：覆盖配置读取、平台匹配、开篇规则和后续章节循环。
