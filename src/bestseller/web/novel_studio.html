BestSeller / 长篇小说控制台

不是“一次性写完一本书”，
而是可控地推进整部长篇。

这个页面直接驱动当前项目的真实写作流水线。你可以先定义题材、卖点、目标体量和一句话设定， 系统会依次完成规划、故事圣经、叙事图谱、章节与场景、审校、修订和导出。对于百万字、千万字项目， 正确做法不是一次性写完，而是按卷、按阶段不断扩写和维护一致性。

当前页面负责什么 发起整书任务、监控阶段进度、查看故事骨架、预览章节正文和导出成品。
长篇如何推进 先做结构，再做单卷，再做整书维护。页面里的总字数和总章节数指的是本轮目标范围。
什么时候该修订 当章节、情绪线、暗线或世界规则没有通过审校时，直接在页面执行自动修复并继续推进。

长篇写作链路

  1. 1
    设定本轮范围
    题材、目标字数、目标章节数、读者定位、一句话 premise。
  2. 2
    生成结构骨架
    规划产物、故事圣经、叙事图谱、章节与场景合同会先建立出来。
  3. 3
    写作与修订
    章节按场景推进，经过审校、自动修订、知识沉淀，再汇总成章。
  4. 4
    导出与继续扩写
    先得到可读草稿，再逐卷扩大范围，而不是一次性把千万字全部跑完。

这里看写作、修复和导出任务的实时阶段，不需要盯着命令行。

已有项目

选择已有项目后，可以继续看结构、正文预览、修订状态和导出文件。

还没有选中项目

先在左侧发起一轮写作，或者打开已有项目查看结构与正文。

项目状态 - 当前整书处于什么阶段，例如规划中、写作中、修订中、已完成。
章节总数 - 当前已物化到数据库里的章节数量。
场景总数 - 场景越多，说明故事推进颗粒度越细。
导出文件 - 当前项目目录下已经生成的可读产物数量。
当前成品字数 - 优先显示整书 `project.md` 的当前正文总字数和预计阅读时长。
世界规则 - 世界观限制、能力边界和运作规则数量。
角色数量 - 故事圣经中当前已建好的核心角色和重要配角数量。
主线 / 支线 - 叙事图谱里已经落地的 plot arcs 数量。
情绪 / 反派线 - 关系线、情绪线与反派推进计划的当前规模。
平台 / 读者承诺 - 当前项目按哪类平台目标和哪种持续期待来组织写作。
流程运行数 - 当前项目历史上已经跑过多少条 workflow，以及最近一条的状态。
